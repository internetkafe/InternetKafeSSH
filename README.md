# InternetKafeSSH — Xray конфиг с маскировкой под SSH

Готовые конфиги Xray (VLESS + TCP + FakeDNS) для создания туннеля, который выглядит как **реальный SSH‑сеанс** на 22‑м порту. Проект родился из экспериментов по обходу DPI — никаких подозрительных гибридов, только чистая имитация SSH.

## Особенности

- **Порт 22** — стандартный SSH‑порт, не вызывающий сомнений у провайдеров.
- **Обмен баннерами** — клиент и сервер обмениваются текстовыми строками `SSH-2.0-OpenSSH_9.9p1 Debian-13\r\n` до начала шифрованного трафика.
- **Минимальные DNS‑следы** — FakeDNS на сервере прячет все DNS‑запросы внутрь туннеля, снаружи ни одного UDP‑пакета к публичным серверам.
- **Гео‑маршрутизация** — Российские домены (geosite:ru) идут напрямую, остальное — через прокси (легко меняется под свою страну).
- **Тонкая настройка таймаутов** — поведение туннеля приближено к обычной SSH‑сессии (policy с коротким handshake).

## Принцип маскировки (что видит DPI)

1. **SYN → SYN-ACK → ACK** — обычное TCP‑рукопожатие на 22‑й порт.
2. **Сервер сразу отправляет пакет**:  
   `SSH-2.0-OpenSSH_9.9p1 Debian-13\r\n` + 16 нулевых байт (имитация cookie)
3. **Клиент отвечает таким же пакетом** с тем же баннером и нулями — DPI фиксирует полноценный обмен идентификационными строками.
4. **Весь дальнейший трафик** — это поток случайных байтов, неотличимый от зашифрованного SSH‑протокола.

Таким образом, инспектор видит классический SSH‑рукопожатие — соединение классифицируется как легитимное и не подвергается дальнейшему анализу.
